Translation Assistant

Full Design Specification — v1.1 | May 2026

Purpose & Scope

An internal, multi-user web application for freelance translators with two core modules:

Freelance Job Management

  • Client & contact management
  • Job tracking with status workflow
  • Invoicing & payment tracking
  • Calendar with deadlines
  • Reporting & analytics

Document Translation

  • Dynamic template creation with multi-page support
  • Block-based document designer
  • Multi-language field labels
  • Google Translate popup helper (per field)
  • Word (.docx) export
  • Clone from existing documents
  • Free-form jobs (upload source + translated files directly)
  • Side-by-side source file viewer

Platform

  • Primary: Web application
  • Planned: Mobile app (React Native)
  • Users: Multiple users, invite-only, RBAC
  • Clients: No client access — internal tool only
  • Deployment: VPS with Docker

Dashed = optional step (can be skipped). A client can call directly and skip quote/accepted. Invoicing is optional.

Pricing Calculation

Formula: calculatedTotal = pageCount × (useDiscountedPrice ? discountedPricePerPage : pricePerPage)

Override: finalPrice — if set, overrides calculatedTotal.

Different currency: If client pays in a non-base currency, paymentCurrency and paymentAmount store the actual amount received.

Multiple language pairs: Each pair = separate job.

File Management

Upload Rules

  • Max upload size: 5MB (configurable in Settings)
  • Allowed/forbidden file types: configurable in Settings
  • Images: auto-compressed on upload using sharp
  • PDFs: best-effort compression of embedded images
  • Other files: stored as-is

Source File Linking

  • Source files can be linked (not copied) from another job to save disk space
  • linkedFromJobId tracks the origin job
  • On job deletion: warn user. If files are linked by other jobs, move ownership to the linked job

Job File Categories

  • Source files: original documents (uploaded or linked from another job)
  • Translated files: uploaded translations (free-form jobs) or generated .docx exports (template-based jobs)
  • Source files are optional — translator may work from physical paper

Workflow — Template Creation

→

  1. Admin creates a new template (e.g., "Birth Certificate"), sets name, description, default price per page
  2. Adds fields: Full Name (text), Date of Birth (date), Place of Birth (text), Father's Name (text), etc.
  3. For each field, enters the label in every defined language (e.g., "Full Name" in English, "الاسم الكامل" in Arabic)
  4. Opens the document designer — arranges fields in rows/columns, adds header text, footer, static text, date placeholders
  5. Saves. Template is ready to use in jobs.

Template-Based:

  1. Create a job — select type "template", client, optional contact, source/target language
  2. Set page count and price (auto-populated from template, can override)
  3. Optionally upload source files or link from another job (for side-by-side reference)
  4. Add one or more documents to the job — each based on a template
  5. Translator fills in each document's fields (with GT popup helper)
  6. Export completed documents to Word (.docx)
  7. Mark job as delivered
  8. Optionally generate an invoice (can cover multiple jobs for same client)
  9. Track payment (or mark as free of charge)

Free-Form:

  1. Create a job — select type "free-form", client, optional contact, source/target language
  2. Set page count and price
  3. Upload source files (or link from another job). Source files are optional.
  4. Translate files externally (on PC, on paper, etc.)
  5. Upload translated files
  6. Mark job as delivered → invoice → paid

Pro Bono Jobs:

Either type can be marked as "Free of Charge" with an optional reason. Price is set to 0, excluded from revenue reports (shown separately), no invoice expected.

Create from Client:

Jobs can be created from the client detail page — the client field is auto-filled.

Workflow — Document Translation

  1. Open a document within a job
  2. Optionally open side-by-side view (split panel: left = source file viewer, right = template form)
  3. See all template fields with label in target language (primary) and source language (small text underneath)
  4. For each field:
    • Type the translation directly (target language) — user references the original document beside them
    • Optionally click GT to open Google Translate popup: enter source text → get translation → copy to field
  5. Reference passport copies (accessible from sidebar) for correct name spellings
  6. Save progress at any time (auto-save recommended)
  7. For multi-page documents: click "+ Add Page" to add additional pages, each with its own fields
  8. When all fields are filled, mark as completed and export

Workflow — Invoice Generation

  1. From a job or from the Invoices section, click "Create Invoice"
  2. Select client (auto-filled if coming from a job)
  3. Add one or more jobs to the invoice
  4. Invoice items are auto-populated from job details (description, pages, rate, total)
  5. Optionally add tax rate
  6. Invoice number is auto-generated (sequential)
  7. Save as draft, then mark as sent
  8. Track payment: mark as paid when payment received
  9. Overdue status applied automatically based on due date

UI Design — Multi-Page Documents

Templates define a page blueprint. When filling a document, the user can add additional pages. Each page has its own fixed fields (e.g., document number) and repeatable entries (e.g., names in a family extract).

Multi-Page Document — Family Extract Example
Page 1 Active
Document Number
455567

2. Sarah Ahmad — 22/07/1988
3. Ali Ahmad — 10/01/1990
Page 2 Remove
Document Number
455566
Entries:
4. Fatima Ahmad — 03/05/1993
5. Khaled Ahmad — 18/11/1995

How it works:

  • Template defines a page blueprint — the default structure for any page
  • Each page has its own fixed fields (e.g., document number — unique per page)
  • Each page has a repeatable zone for entries (e.g., names, dates)
  • User clicks "+ Add Page" to add pages — each new page uses the same blueprint
  • User clicks "+ Add Entry" within a page to add entries to that specific page
  • On export, each page generates as a separate page in the Word document
  • The number of entries per page matches the original document exactly

Security

Authentication

JWT + refresh token rotation. Tokens stored in httpOnly cookies. Short access token TTL (15 min), long refresh TTL (7 days). Bcrypt password hashing.

Authorization

RBAC enforced on every API endpoint. Permission checks at controller level via guards. Job-level sharing with explicit permission grants (view/edit).

Input Validation

All inputs validated with class-validator (NestJS). Parameterized queries via ORM — no raw SQL. File upload validation (type, size limits).

API Security

Helmet for HTTP headers. CORS restricted to app domain. Rate limiting on auth endpoints. CSRF protection for cookie-based auth.

Data Protection

Sensitive fields encrypted at rest. Passport copies stored outside web root. File access through authenticated API only — no direct URL access.

Audit & Logging

All auth events logged. Failed login tracking. API request logging (sanitized — no passwords/tokens in logs).
